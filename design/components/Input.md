# Input

Text / password input field. Lives inside a `.card` with a required
label. Accessible by design — label is always associated with the input
via `for`/`id`.

The live API table is in the **Docs** tab of `Components/Input`. This
page focuses on matrices and guidelines.

## Props

```ts
type InputProps = {
  label: string;                          // REQUIRED (a11y)
  placeholder?: string;
  value?: string;
  type?: 'text' | 'password';             // default 'text'
  actionLabel?: string;                   // inline action button
  disabled?: boolean;                     // freeze during async ops
};
```

## Matrix — type × content state

| Type     | Empty | Filled | With action | Disabled |
|----------|-------|--------|-------------|----------|
| text     | placeholder | value | action visible | faded + disabled attr |
| password | placeholder | `••••••` mask | "Show" / "Hide" typical | faded + disabled attr |

## Matrix — interaction states

| State    | Behavior                     |
|----------|------------------------------|
| Default  | transparent border           |
| `:hover` | _not customized_             |
| `:focus` | `border-color: accent gold`  |
| `:focus + :hover` | same as `:focus`    |

`:active` doesn't apply to inputs.

## Accessibility

- **Label required** — `label` prop is not optional. Screen readers need
  a label to announce the input's purpose. No visually-hidden-label
  variant (would mask potential issues).
- **Label/input association** — Input renders `<label for="id">` +
  `<input id="id">`. The id is auto-generated by the component.
- **Placeholder is NOT a label substitute** — placeholder disappears on
  focus and is grey-colored (low contrast). Never rely on placeholder
  alone to explain what a field is for.

## Guidelines

- **One Input per `.card`** is the default. For multi-field forms, stack
  multiple `.card > .inp-group` — the spacing reads cleanly.
- **`actionLabel` is an action, not a button** — keep the label short
  (one verb, ≤ 10 chars). Examples: "Show", "Hide", "Edit", "Save".
  For destructive or navigation actions, use a separate Button below the
  Input, not an inline action.
- **`type: 'password'`** triggers the OS password-manager affordance and
  native masking. Don't try to reimplement password masking — let the
  browser handle it.
- **`disabled` for async states** — when an API call is in flight (WiFi
  connection, saving settings), disable the input until the response.
  Avoid disabling without a visible reason (spinner in the submit button,
  context message, etc.).

## Related

- `Button` — typically sits below or alongside an Input for form
  submission.
- `Card` — the surface primitive that wraps Input(s).
